You are an AI systems engineer and runtime auditor.

Goal: Perform a full internal audit of this Agentic AI project before production deployment.

Instructions:
1. **Environment Check**
   - Verify Python version, environment variables, and virtual environment consistency.
   - List installed dependencies (`pip list`) and compare with `requirements.txt`.
   - Detect missing or conflicting packages (e.g., langchain-core, openai, llama-index, crewai, smolagents).
   - Ensure `venv` or `poetry` isolation is properly configured in Replit.

2. **Project Structure Validation**
   - Map the folder hierarchy: `/src`, `/agents`, `/memory`, `/tools`, `/config`.
   - Check that each module has an `__init__.py` if needed.
   - Detect any circular imports or unresolved references.

3. **Agent Configuration Audit**
   - Inspect all agent classes and YAML/JSON configs under `/agents`.
   - Confirm each agent has: role definition, tools list, input/output schema, and execution policy.
   - Flag any duplicated or missing agent identifiers.
   - Verify tool binding correctness (function names, input signatures).

4. **Runtime and Orchestration Layer**
   - Identify the main controller (e.g., `manager.py`, `main.py`, `crew.py`).
   - Trace the message-passing or task delegation pipeline (event loop, queue, async scheduling).
   - Check for concurrency issues (e.g., missing await, blocking I/O, or multi-agent deadlocks).

5. **Memory & Context Integrity**
   - Ensure context persistence (vector store, Redis, SQLite, etc.) is configured and accessible.
   - Validate embedding model consistency (same model used for storage and retrieval).
   - Detect memory leakage or stale cache files.

6. **API & Keys**
   - Check `.env` for correct variable usage (e.g., OPENAI_API_KEY, LANGCHAIN_API_KEY).
   - Ensure keys are not hardcoded anywhere in the repo.

7. **Testing & CI/CD Readiness**
   - Run unit tests if `/tests` exists.
   - Log all failed imports, undefined agents, or deprecated LangChain/LLM API calls.

8. **Final Summary Report**
   - Output a categorized diagnostic summary:
     ✅ Passed modules
     ⚠️ Warnings (config mismatch, missing init)
     ❌ Critical errors (import failure, undefined agent, missing key)
   - Suggest exact next steps to fix each critical issue.
